BIM Pipeline Application

Enterprise BIM workflow orchestration · 2024 – Present · Bechtel

The Problem

Multi-site BIM projects require constant exports, format conversions, and health checks across half a dozen disconnected enterprise CAD platforms. The default workflow is manual, error-prone, license-hungry (every user needs every seat), and doesn't scale past a handful of sites.

What It Is

A multi-software orchestration platform that runs the entire BIM pipeline unattended on Windows service accounts. A manager clicks one button; exports, conversions, and copies run overnight across multiple VMs and report back when they're done.

7+ CAD platforms orchestrated
1 service-account license per VM (vs. per-user)
Multi‑site deployed and running across Bechtel project servers

Highlights

  • Deployed and running unattended across multiple Bechtel project servers; self-locating paths.json config enables single-file deployment to any new project site.
  • WinForms application with modular page architecture — Dashboard, Flows, Server Health, File Health, Setup, and Options — with structured JSON logging and live heartbeat monitoring across all managed VMs.
  • Coordinates Tekla, Revit, SmartPlant 3D, Navisworks, AutoCAD, iConstruct, and PlantSight via service-account execution on multiple Windows VMs.
  • Resolved complex infrastructure constraints: DCOM configuration for service-account COM access, Session 0 process isolation, SPIOP/Intergraph Batch Manager license behavior, Roamer.exe batch invocation for Navisworks NWD exports, and SQL Server permissions for service accounts.

Volunteer

The Problem

SNAP-Ed, a federally funded nutrition education program, relied on manual processes to sort ambassador data, build event calendars, maintain archives, and audit submissions for formatting errors. The work was time-consuming, repetitive, and had no dedicated owner — it simply piled up.

What It Does

A data automation tool that ingests ambassador reporting data, sorts and organizes it automatically, generates formatted event calendars, maintains an archived calendar history, and audits the dataset for formatting compliance — flagging errors before they propagate downstream.

~30 hours of manual labor eliminated per cycle
$0 cost to the nonprofit — built and delivered pro bono

Highlights

  • Calendar generation produces formatted event schedules and maintains a structured archive of past calendars — no manual assembly required.
  • Built-in audit logic checks submissions for formatting compliance and surfaces errors before they reach downstream reporting.
  • Delivered to a nonprofit with no technical staff — designed to run without ongoing maintenance or engineering support.
